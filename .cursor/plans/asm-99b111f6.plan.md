# Full-Stack Quiz App Plan (Assignment 4)

## Phase 0 — Scope & Setup

- Confirm stack: Node/Express + MongoDB + React + Redux + React Router + Bootstrap 5.
- Initialize repos/workspaces; create `.env` templates (PORT, MONGO_URI, JWT_SECRET).

## Phase 1 — Backend Foundations

- Scaffold Express app with ES modules or CJS consistently; add middlewares (JSON, CORS, morgan, error handler).
- Connect to MongoDB via Mongoose; seed minimal admin/user accounts for demo.
- Define models: `User`(roles), `Quiz`, `Question`(referenced by Quiz), `Article`.
- Implement auth: signup/login routes issuing JWT; middleware for `auth` and `requireAdmin`.
- CRUD APIs:
- Quiz: create/read/update/delete; include population of questions for read.
- Question: create/read/update/delete scoped to quiz.
- Article: admin CRUD; public read.
- Validation & errors: request validation (e.g., Joi/express-validator) and unified error responses.
- Testing: minimal route smoke tests or Postman collection; sample data seeding script.

## Phase 2 — Frontend Foundations

- Scaffold React app; add Redux Toolkit, React Router, axios, Bootstrap 5.
- Set up store slices: `auth`, `quizzes`, `questions` (per quiz), `articles`, `ui/status`.
- Configure axios baseURL and auth header injection from token.
- Layout with shared navbar; protected routes for user and admin areas.

## Phase 3 — Auth Flows

- Build pages: Login, Register; store JWT + user info; handle logout.
- Guard routes: user dashboard/quiz pages require auth; admin routes require role `admin`.

## Phase 4 — User Quiz Experience

- Fetch quizzes after login; list/select quiz.
- Quiz detail page: render questions/options, radio selection, submit; compute score locally or via API; show result and restart.
- Basic article list/read view for users.

## Phase 5 — Admin CRUD UI

- Admin dashboard with tabs: Home, Manage Questions, Manage Quizzes, Manage Articles.
- CRUD forms + tables/cards:
- Quiz management: create/update/delete quiz metadata and question associations.
- Question management: create/update/delete questions/options/correct index.
- Article management: create/update/delete articles.
- Use Redux thunks for all mutations; optimistic or refetch-based updates.

## Phase 6 — Polishing & Validation

- Form validation and user-friendly error toasts; loading states; empty states.
- Bootstrap styling to match/beat reference UI.
- Accessibility: labels/aria, focus on submit, keyboard-friendly.

## Phase 7 — Testing & Handover

- Manual test matrix: auth, role-guard, CRUD flows, quiz submit, restart, API errors.
- Lint/format, tidy `.env.example`, README with setup/run instructions and default accounts.
